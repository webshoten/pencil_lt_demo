## 大嶋さんのAI経歴

- 2024年5月頃〜: GitHub Copilot（VSCode）を使い始める
  - 主にコード補完（単語予測的な使い方）、Markdown化くらい
  - 本来の力を使いこなせてなかった
  - 【ネタ】当時のCopilotの実力 vs 自分の使い方 の対比が面白い
    - 実力: Chat、コード説明、テスト生成、バグ修正提案など
    - 自分: 単語補完とMarkdown化だけ
- 2025年2月〜: 現PJ参画、AI推奨環境
  - 最初の3ヶ月はAIに頼らず補助的に使う程度
    - 理由: 頼りすぎると自分の理解が止まる気がしていた
    - 【振り返り】AIなしだと手が止まるなら理解は止まってるとも言える
    - 一方でAIを使うことで知識は圧倒的に増えた
    - 理解の形が変わった、という方が正確かも
    - 【核心】何が減って何が増えたのかを理解することが大事
      - 減ったもの: ゼロから作り出す思考力（ただし元々テンプレから作ってたので大差ないかも）
      - 増えたもの: 圧倒的な知識量、設計・実装スピード、バグ対処能力
    - 【データ】contribute数は意外と変わっていない（グラフあり）
      - Issue対応スピードは上がったがコミット数は大差なし
      - 量ではなく質や余裕が変わった？
      - 理由は本人もわからない → LTで正直に言うのもアリ
    - AIに任せない領域
      - 自分自身の仕様理解
      - 仕様・品質の決定権
  - 途中からv0（Vercel）を許可得て使い始める
    - 要件→叩き台デザインは作れた
    - ただし一方通行。既存コードベースからの生成は難しかった
- 2025年6月頃〜: Cursor使い始める
  - コード全体を理解してくれるので話が噛み合う
  - v0と違い既存コードベースの文脈で会話できる
  - モデルはClaude（Cursor内蔵）を使っていた
  - ただし自分の使い方がまだ確立しておらず手探り状態
  - Composer登場後、勝手に実装される・意図しない変更が増えて悩み増加
  - ruleで制御しようとすると→マイクロマネジメント化して逆に非効率
    - 細かいこと全部聞いてくるようになってしまう
  - 【ネタ】AIとの距離感問題: 自由にさせすぎると暴走、縛りすぎると非効率
  - これはPLがメンバーに対して感じる悩みと似ている
    → AIへの指示 ≒ チームマネジメント
  - スピードと品質のバランスをマネジメントする必要に気づく
    - 細かい実装はAIに任せて、テストコードや振る舞いで品質を担保
    - コードは後から確認するスタイルに変化
    - 「全部理解してから進める」→「動作で担保して後追い理解」へシフト
- 2025年後半〜: Claude Code CLIへ移行
  - 個人で触る機会があり、CLIに違和感なかったのでそのまま移行
  - Cursor + Claudeが重かった（LSP起因？）のも理由
  - 特に大きなきっかけはなく自然な流れ
  - Claude Code CLIにしてからIssueを並列対応できるようになった
    - ターミナル複数起動で同時進行
  - コード理解のアプローチが変わった
    - Before: 動かす→ブレークポイント/ログで把握
    - After: AIに仕様把握→周辺コード把握→自分で確認→計画立てる
    - デバッグ的アプローチからレビュー的アプローチへ
  - AIが見れる範囲を広げるためにMCPも導入
    - Chrome MCP → 動いたがそれほど使えず
    - DB参照用のローカルMCPを自作 → 重くなる＋DBをAIに触らせる怖さもあり断念
    - drawio MCP → 良かった
    - Pencil（pencil.dev）+ Pencil MCP → 非常に良かった
      - IDE内でデザインできるキャンバスツール
      - MCP経由でClaude Codeがキャンバスを読み取り→コード生成
      - デザイン→コードのギャップを埋める
      - 良かった点:
        - push後にデザインの差分をcursorで確認できる
        - デザイン設計をAIにやってもらえる
    - MCPは試行錯誤中（当たり外れあり）

## Pencil導入手順

### Step 1: Pencilアプリのインストール
- https://www.pencil.dev/ から macOS用 `.dmg` をダウンロード
- アプリケーションフォルダにドラッグ
- 起動してメールでアクティベーション

### Step 2: Claude Code CLIのインストール（未導入の場合）
- `npm install -g @anthropic-ai/claude-code-cli`
- `claude` コマンドで認証フローを完了
- `claude --version` で確認

### Step 2.5: VSCode / Cursor拡張のインストール（補足）
- VSCode Marketplace で「Pencil」を検索（ID: `highagency.pencildev`）
- Cursorでも同様にインストール可能
- IDE内でデザインキャンバスが使えるようになる

### Step 3: MCPの接続
- Pencilを起動するとMCPサーバーが自動起動（設定不要）
- Pencilを開いた状態でClaude Codeを起動する（順序が大事）
- Cursorの場合: 設定 → ツール＆MCP で「Pencil」が表示されることを確認

### 利用可能なMCPツール
| カテゴリ | ツール | 機能 |
|---------|--------|------|
| 設計 | batch_design | 要素作成・修正・削除 |
| 設計 | batch_get | コンポーネント構造の読取 |
| 分析 | get_screenshot | デザイン検証用プレビュー |
| 変数 | get/set_variables | デザイントークン管理 |
